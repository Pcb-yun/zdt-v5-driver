# ZDT_X42S 第二代闭环步进电机通讯指令

本文档提取自ZDT_X42S第二代闭环步进电机用户手册第5章，整理了所有可向电机发送的通讯指令及电机返回信息的格式。

---

## 一、基础说明

### 1.1 命令后缀说明
- 命令后缀带有**（X）**为X固件命令
- 带有**（E）**为Emm固件命令
- 无后缀以及带有**（X42S/Y42）**都是两种固件通用命令

### 1.2 两种固件部分命令差异

| 命令 | 差异说明 |
|------|----------|
| 1.力矩模式 | Emm固件无力矩模式，X固件有 |
| 2.速度模式 | 命令格式不同 |
| 3.位置模式 | 命令格式不同，X固件有两种位置模式 |
| 4.读取系统状态参数 | 命令格式不同 |
| 5.修改系统状态参数 | 命令格式不同 |
| 6.读取驱动配置参数 | 主机发送的命令格式一样，电机返回格式不同 |
| 7.修改驱动配置参数 | 命令格式不同 |
| 8.存储一组速度参数，上电自动运行 | 命令格式不同 |
| 9.电机位置角度/误差 | 两种固件的命令中电机位置角度/误差的格式不同 |

除上述命令外，两种固件其他命令格式相同

### 1.3 通用返回码说明

| 返回码 | 含义 |
|--------|------|
| 02 | 表示接收的命令正确 |
| E2 | 表示接收的命令参数错误（数据范围不满足或条件不满足，如触发低压警告、堵转保护、过流过热保护等） |
| EE | 表示接收的命令格式错误 |
| 9F | 表示接收动作执行完成命令（电机主动返回） |

## 三、运动控制命令

### 3.1 多电机命令（X42S/Y42）

**主机发送（主机→电机）**

| 字节1 | 字节2 | 字节3 | 字节4 | 字节n |
|-------|-------|-------|-------|-------|
| 地址 00 | 功能码 AA | 总字节数 | 命令1 | 命令2...命令n | 校验码 6B |

**电机返回（电机→主机）**

| 字节1 | 字节2 | 字节3 | 字节4 |
|-------|-------|-------|-------|
| 地址 Addr | 功能码 | 返回数据 | 校验码 6B |

**说明**：
- 所有运动命令只有地址1电机会回复确认收到命令
- 多电机命令中，位置模式运动命令中的多机同步标志用于控制是否返回到位命令
- Modbus-RTU协议暂不支持多电机命令
- **与多机同步运动的区别**：多电机命令（0xAA）是X42S/Y42独有的**打包发送**功能，将多个电机的命令合成一帧发送；多机同步运动（0xFF 66，见3.16节）是**通用功能**，所有型号都支持，用于触发已缓存命令的电机同步执行

多机同步运动需配合运动命令中的**同步标志**使用：
1. 先向各电机发送运动命令，同步标志设为**01**（缓存不执行）
2. 再用广播地址发送 `00 FF 66 6B` 触发同步
3. 所有缓存了命令的电机同时开始运动，无缓存的电机不动作

> 例如：地址1发送位置模式命令（同步标志=01），地址2发送速度模式命令（同步标志=01），然后广播发送 `00 FF 66 6B`，两电机同时开始运动。

**适用范围**：
- **所有型号通用**：X42_V1.3、X42S、Y42 均支持
- **所有固件通用**：X固件和Emm固件均支持
- 仅要求运动命令带有同步标志参数

**与多电机命令的区别**：
| 功能 | 多电机命令（0xAA） | 多机同步运动（0xFF 66） |
|------|-------------------|------------------------|
| 作用 | 一帧打包发送多个命令 | 触发已缓存的命令同步执行 |
| 支持型号 | **仅X42S/Y42** | **所有型号** |
| 通讯效率 | 高（一次发送多个命令） | 低（需逐条发送再触发） |
| 依赖关系 | 独立功能 | 需配合同步标志=01使用 |

**示例**：发送 `00 FF 66 6B`，正确返回 `01 FF 02 6B`

| 参数 | 范围 | 默认值 | 单位 | 说明 |

| 回零模式 | 00-05 | 00(单圈就近回零) | - | 见下方模式对照表 |
| 回零方向 | 00/01 | 00(CW) | - | 00=CW，01=CCW |

**回零模式对照表**：

| 值 | 模式名称 | 说明 |
|----|----------|------|
| 00 | 单圈就近回零 | 向当前位置就近的方向回零 |
| 01 | 单圈方向回零 | 按设定的回零方向回零 |
| 02 | 无限位碰撞回零 | 无需限位开关，碰撞堵转后作为零点 |
| 03 | 限位回零 | 通过限位开关信号触发回零 |
| 04 | 回到绝对位置坐标零点 | 回到编码器记录的绝对位置零点 |

**碰撞回零原理**：

碰撞检测将电机堵转位置作为零点，需同时满足以下三个条件：
(1) 电机实时转速 < 碰撞回零检测转速
(2) 电机实时相电流 > 碰撞回零检测电流
(3) 以上两个条件持续的时间 > 碰撞回零检测时间

**返回数据解析示例**：

| 字节序列 | 值 | 含义 |

| 00 | 00 | 回零模式=Nearest(单圈就近回零) |
| 00 | 00 | 回零方向=CW |
| 00 1E | 0x1E00=30 | 回零速度=30RPM |
| 00 00 27 10 | 0x10270000=10000 | 回零超时时间=10000ms |
| 01 2C | 0x2C01=300 | 碰撞回零检测转速=300RPM |
| 03 20 | 0x2003=800 | 碰撞回零检测电流=800mA |
| 00 3C | 0x3C00=60 | 碰撞回零检测时间=60ms |
| 00 | 00 | O_POT_En=不使能 |
| 6B | 6B | 校验码 |

### 5.2 读取固件版本和硬件版本

**主机发送（主机→电机）**

| 字节1 | 字节2 | 字节3 |
|-------|-------|-------|

**电机返回（电机→主机）**

| 字节1 | 字节2 | 字节3 | 字节4 | 字节5 | 字节6 | 字节7 |
|-------|-------|-------|-------|-------|-------|-------|
| 地址 Addr | 功能码 1F | 固件版本 | 硬件系列 | 硬件类型 | 硬件版本 | 校验码 6B |

**返回数据解析流程（0x1F）**

电机返回共7字节，格式如下：

| 字节偏移 | 内容 | 说明 |
|----------|------|------|
| 0 | 地址 Addr | 电机地址 |
| 1 | 功能码 0x1F | - |
| 2-3 | FW_Ver（bit15-0） | 固件版本号，高字节在前，如0x00C8=200=V2.0.0 |
| 4 | HW_Series（bit7-4） | 硬件板卡系列：0=X系列，1=Y系列 |
| 4 | HW_Type（bit3-0） | 硬件板卡类型：0/1/2/3/4/5/6=20/28/35/42/57/86 |
| 5 | HW_Ver（bit7-0） | 硬件版本号，如0x14=20=V2.0 |
| 6 | 校验码 | 6B/XOR/CRC-8 |

> 字节4为复合字节：高4位为硬件系列，低4位为硬件类型。
> 例如字节4=0x32，则HW_Series=3(X系列)，HW_Type=2(35型号)。

**版本号解读规则**：

| 字段 | 解读方式 | 示例 |
|------|----------|------|
| FW_Ver | 高8位=主版本，低4位=次版本，最低4位=修订号 | 200=V2.0.0，121=V1.2.1 |
| HW_Ver | 高4位=主版本，低4位=次版本 | 20=V2.0，14=V1.4 |

**解析示例**：

发送 `01 1F 6B`，假设返回 `01 1F C8 00 32 14 6B`

| 字节序列 | 值 | 含义 |
|----------|------|------|
| 01 | 01 | 地址 |
| 1F | 1F | 功能码 |
| C8 00 | 0xC800=51200 | 固件版本=V2.0.0（注：0x1F固件版本字段为特殊编码，非标准大端序） |
| 32 | 0x32 | HW_Series=3(X系列)，HW_Type=2(35型号) |
| 14 | 0x14=20 | 硬件版本=V2.0 |
| 6B | 6B | 校验码 |

> 与0x1A（选项参数状态）的区别：0x1A读取的是可修改的运行配置（控制模式、方向等），0x1F读取的是出厂固定的硬件身份信息（固件版本、硬件型号等）。

### 6.4 读取选项参数状态（X42S/Y42）

**主机发送（主机→电机）**

| 字节1 | 字节2 | 字节3 |
|-------|-------|-------|
| 地址 Addr | 功能码 1A | 校验码 6B |

**电机返回（电机→主机）**

| 字节1 | 字节2 | 字节3 | 字节4 |
|-------|-------|-------|-------|
| 地址 Addr | 功能码 1A | 状态标志 | 校验码 6B |

**状态标志位说明**：

**第1字节（bit0-bit7）**：

| bit | 名称 | 说明 |
|-----|------|------|
| bit0 | MotType | 电机类型，0=1.8°步进电机，1=0.9°步进电机 |
| bit1 | FwType | 固件类型，0=X固件，1=Emm固件 |
| bit2 | CtrMode | 控制模式，0=开环，1=闭环 |
| bit3 | 保留 | - |
| bit4 | MotDir | 电机运动方向，0=CW，1=CCW |
| bit5 | BtLock | 按键锁定状态 |
| bit6 | 保留 | - |
| bit7 | Scale | 缩小10倍输入功能 |

**第2字节（bit8-bit15）**：

| bit | 名称 | 说明 |
|-----|------|------|
| bit8-bit9 | LockLevel | 锁定修改参数等级，00=解锁，01=禁止修改通讯参数，10/11=禁止修改所有参数+触发校准 |
| bit10-bit15 | 保留 | - |

**锁定参数等级详细说明**：

| bit8-bit9 | 等级 | 说明 |
|-----------|------|------|
| 00 | 等级0 | 解锁，可修改任意参数 |
| 01 | 等级1 | 禁止修改（ID地址+通讯速率+通讯协议+通讯端口复用） |
| 10 | 等级2 | 禁止修改所有参数+触发校准 |
| 11 | 等级3 | 禁止修改所有参数+触发校准 |

**代码判断示例**：
```c
// 读取锁定等级
uint8_t lock_level = (byte2 >> 0) & 0x03;
// 读取按键锁定状态
bool bt_lock = (byte1 >> 5) & 0x01;
```

**各字段读取方式说明**：

0x1A返回的各字段中，**均无单独的专用读取指令**，读取方式如下：

| bit | 字段 | 读取方式 | 说明 |
|-----|------|----------|------|
| bit0 | 电机类型 | **仅0x1A** | 无其他读取方式 |
| bit1 | 固件类型 | **仅0x1A** | 无其他读取方式 |
| bit2 | 控制模式 | 0x1A 或 0x42（X固件第2个参数） | Emm固件只能通过0x1A |
| bit4 | 电机运动正方向 | **仅0x1A** | 无其他读取方式 |
| bit5 | 按键锁定状态 | 0x1A 或 0x42（X固件第1个参数） | Emm固件只能通过0x1A |
| bit7 | 缩小10倍输入 | 0x1A 或 0x42（X固件第19个参数） | Emm固件只能通过0x1A |
| bit8-bit9 | 锁定参数等级 | **仅0x1A** | 无其他读取方式 |

> 电机类型、固件类型、运动正方向、锁定参数等级**只能通过0x1A读取**，是获取这些信息的唯一方式。

**与其他读取指令的覆盖关系**：

| 字段 | 0x1A | 0x42（X固件） | 0x42（Emm固件） | 单独读取 |
|------|------|---------------|----------------|---------|
| bit0 电机类型 | ✓ | ✗ | ✓（第1个参数） | ✗ |
| bit1 固件类型 | ✓ | ✗ | ✗ | ✗ |
| bit2 控制模式 | ✓ | ✓（第2个参数） | ✗ | ✗ |
| bit4 运动正方向 | ✓ | ✗ | ✗ | ✗ |
| bit5 按键锁定 | ✓ | ✓（第1个参数） | ✗ | ✗ |
| bit7 缩小10倍输入 | ✓ | ✓（第19个参数） | ✗ | ✗ |
| bit8-bit9 锁定等级 | ✓ | ✗ | ✗ | ✗ |

> 注：0x1A与0x43（系统状态参数）**无任何重复**；与0x42的重复：电机类型仅Emm固件，控制模式/按键锁定/缩小10倍输入仅X固件。

## 附录：控制命令应答方式

| 应答方式 | 说明 |
|----------|------|
| 00 (None) | 发送命令后电机不返回任何信息 |
| 01 (Receive) | 只返回确认收到命令（地址+功能码+02/E2/EE+6B） |
| 02 (Reached) | 只返回到位/回零完成/夹爪夹紧命令（地址+功能码+9F+6B） |
| 03 (Both) | 既返回确认收到命令，也返回动作完成命令 |
| 04 (Other) | 位置模式返回动作完成命令，其他控制命令返回确认收到 |

## 附录：多字节参数字节序说明

电机通讯协议中，所有多字节参数均采用**大端序（Big-Endian）**，即**高字节在前、低字节在后**。

**示例解析**：
- 总线电压字节序列 `5C 67` → 实际值 = `0x5C67` = 23655(mV)
- 总线电流字节序列 `00 00` → 实际值 = 0(mA)
- 相电流字节序列 `00 03` → 实际值 = 3(mA)
- 编码器原始值字节序列 `30 B5` → 实际值 = 12469
- 线性化编码器值字节序列 `43 EB` → 实际值 = 17387
- 目标位置字节序列 `00 01 00 00` → 实际值 = 0x00010000 = 65536

**特别说明**：
- 8.1节 读取系统状态参数（X）和8.2节 读取系统状态参数（Emm）的返回数据中，多字节参数均遵循大端序规则解析。
- 运动控制命令中的位置角度、速度、加速度等参数在主机发送时，也应遵循大端序字节排列。
